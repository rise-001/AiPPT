---

Banana Slides 使用 Markdown 文件（`.md` / `.txt`）导入导出大纲和页面描述，格式可读性强，可用任意文本编辑器编辑。

## 格式结构

```markdown
# 项目标题

> 生成时间: 2025/1/1 12:00:00

---

## 第 1 页: 封面

> 章节: 引言

这一页作为封面，只展示演示主题、公司名称和日期。

**页面描述：**
--- 页面文字 ---

### 项目年度汇报

公司名称

2026 年 4 月

--- 页面文字结束 ---

排版布局：标题居中，大字号，副标题和日期置于下方

---

## 第 2 页: 市场概况

> 章节: 分析

这一页概述行业增长趋势、主要细分市场和关键机会。

**页面描述：**
--- 页面文字 ---

### 市场概况

- 行业规模持续增长
- 企业级需求增速更快
- 细分市场仍存在空白机会

--- 页面文字结束 ---

视觉元素：增长柱状图、关键数据卡片、市场细分示意图

---
```

## 字段说明

| 字段 | 语法 | 必填 |
|------|------|------|
| 页面标题 | `## 第 N 页: 标题` 或 `## Page N: Title` | 是 |
| 章节 | `> 章节: 名称` 或 `> Chapter: Name` | 否 |
| 大纲概述 | 页面标题后的一句普通文本 | 否 |
| 页面描述 | `**页面描述：**` 或 `**Page Description:**` 后使用 `--- 页面文字 ---` / `--- 页面文字结束 ---` 包裹页面正文 | 否 |

如果描述末尾包含形如 `视觉元素：...`、`排版布局：...` 的额外字段，导入时也会一并保留。

页面之间用 `---` 分隔。

## 导出选项

可从大纲编辑器和详细编辑器导出：

| 来源 | 内容 |
|------|------|
| 大纲编辑器 | 页面标题和大纲内容 |
| 详细编辑器 | 可导出「仅描述」或「大纲 + 页面描述」 |

## 导入行为

- 导入的页面会**追加**到现有项目，不会替换已有页面。
- 当前推荐格式是在页面标题后直接写一句大纲概述；旧的 `**大纲要点：**` / `- 要点` 结构仍可导入。
- 同时兼容中英文标记：`第 N 页` / `Page N`、`章节` / `Chapter`、`大纲要点` / `Outline Points`、`页面描述` / `Page Description`。
- 导入内容中的 HTML 标签会被自动清除。
- 页面描述末尾的短字段行（如 `视觉焦点：...`）会被识别为额外字段并保留。
- 支持 `.md` 和 `.txt` 文件。

## 最简示例

一个有效的导入文件可以很简单：

```markdown
## 第 1 页: 开场介绍

这一页介绍演示主题、议程和演讲者。

## 第 2 页: 总结

这一页回顾核心结论和后续行动。
```

## 用外部 AI 生成导入文件

如果你想先在外部 AI 助手中批量整理大纲，或让它按固定结构产出可导入文本，可以使用下面的模板。

将提示词复制到任意 AI 助手（如 ChatGPT、Claude、Kimi），替换占位符后，让它只输出 Markdown 结果，再导入 Banana Slides。

<Tip>
  这只是给外部 AI 生成导入文件的辅助方式，不是 Banana Slides 的主要创作入口。多数情况下，直接在应用内创建项目并使用内置 AI 生成会更方便。
</Tip>

<Tip>
  这里的“页面描述”不是一句简短的画面说明，而是该页的主要文字内容与设计要求的组合。通常做法是：
  先在 `**页面描述：**` 下写该页会直接展示的 Markdown 文本，再按需在末尾补充 `视觉元素：...`、`视觉焦点：...`、`排版布局：...`、`演讲者备注：...` 这类字段。
</Tip>

<Tip>
  如果只需要大纲（不含描述），从提示词中删除 `**页面描述：**` 相关要求即可。若想输出英文标记，也可将 `第 N 页 / 章节 / 大纲要点 / 页面描述` 换成 `Page N / Chapter / Outline Points / Page Description`。
</Tip>

<Tip>
  当前 Banana Slides 内置 prompt 更接近下面这种结构：
  页面标题之后直接给一句简短概述，不再单独生成 `**大纲要点：**`；而在 `**页面描述：**` 内部，`--- 页面文字 ---` / `--- 页面文字结束 ---` 现在应视为必需结构，而不是可选写法。
</Tip>

<Tip>
  直接导入 Markdown 时，系统仍兼容旧的 `**大纲要点：**` / `- 要点` 结构；新内容建议使用标题后的简短概述句。
</Tip>

贴近当前内置 prompt 的示例页更接近下面这种写法：

```markdown
## 第 3 页: 市场机会

> 章节: 行业分析

这一页说明市场规模增长、竞争格局分散，以及企业级机会正在放大。

**页面描述：**
--- 页面文字 ---

### 市场机会正在快速放大

- 全球市场规模预计在 2028 年达到 **320 亿美元**
- 企业级场景增速高于消费级场景
- 当前竞争格局分散，头部集中度仍然较低

| 维度 | 当前现状 | 机会判断 |
| --- | --- | --- |
| 市场规模 | 高增长 | 值得进入 |
| 竞争强度 | 中等 | 可差异化 |
| 付费意愿 | 明确 | 可快速验证 |

--- 页面文字结束 ---

视觉元素：增长曲线、对比数据卡片、行业细分示意图
视觉焦点：320 亿美元、企业级增速、竞争格局分散
排版布局：上方标题，左侧要点，右侧数据卡片，下方表格
```
